# 传统对齐与方法边界

本文记录“命相卜”各模块的主要算法口径、复核依据与已知简化项，便于传统术数工作者、研究者和开发者审阅。

> 本文只说明软件实现边界，不为任何术数体系的有效性背书，也不代替师承判断或专业意见。

## 总体原则

项目将输出分为三层：

1. **盘面事实**：由固定算法生成的干支、星位、宫位、卦爻、历法或地占结构；
2. **传统象义**：依据当前体系提供的固定解释与候选判断；
3. **现实判断**：必须由使用者结合现实证据、处境和专业意见完成。

可核验规则尽量通过古籍课例、立成表、历法锚点和公开算法复核。存在流派分歧、文献不足或简化实现时，界面使用“候选”“推定”“实验”“待考”等标记，不把程序结果冒充完整人工断盘。

## 命类

### 八字四柱与韩国四柱

- 支持子时换日三种口径：子初换日、早晚子时分派、子正民用换日。
- 旺衰以得令、得地、得势三层启发式评分；人元司令另列供参，不直接计入得令分。
- 通根区分本气、中气、余气，并考虑冲支减力。
- 主格参考月令取用逻辑，普通格局仅给候选；从格、专旺等需要人工复核。
- 调候依据十天干十二月表列候选，与扶抑并列，不把单一表格视为最终用神。
- 支持出生地 UTC 偏移、经度和真太阳时校正；未启用时盘面会提示。
- 韩国四柱使用同一历法基础，并按出生地时区与经度处理，不静默套用中国时区。

已知边界：

- 未完成通关、病药、完整成败相神与所有特殊格局的自动取舍。
- 程序旺衰分数是可解释启发式，不等同传统师承中的最终强弱、格局或喜忌结论。

### 紫微斗数

- 实现十四主星与主要辅煞星的庙旺利陷、长生十二神、小限、斗君及多组常见杂曜。
- 支持指定太岁年的流年四化、流曜、岁前与将前十二神。
- 可选流月、流日、流时；闰月提供不同流派选项。
- 当前四化表采用通行南派口径，庚、壬天干等他派分歧会在盘面标注。

已知边界：

- 尚未覆盖所有杂曜异派表与完整现代流盘细法。
- 出生时区、经度和真太阳时尚未完整接入紫微模块。
- 宫干飞化、自化及闰月等规则存在派别差异，不能视为唯一正统口径。

### 西方占星与吠陀占星

- 西方占星包含主要星体落座、上升、天顶、整宫制宫位、古典尊贵、Egyptian bounds 与 Chaldean decans。
- 只有提供出生地经纬度时才输出上升与宫位。
- 吠陀占星采用 Lahiri 恒星黄道，包含 Lagna、D7/D9/D10/D12 分盘、Vimshottari 大运子运、经典瑜伽和 Ashtakavarga。
- Shadbala 当前实现 Naisargika、Uccha、Dig 等确定性较高的项目。

已知边界：

- 行星位置为角分级近似，靠近星座边界时会提示复核。
- Shadbala 的 Kala、Cheshta 等精细项尚未完整实现。
- Ashtakavarga 的同主缩减口径存在分歧；Prastara、完整行运与 Jaimini 体系尚未覆盖。

### 藏历占星

- 使用浦派 Phugpa 历算，处理真月、重日、缺日、闰月和洛萨。
- 生年以洛萨为界，因此可能与农历生肖年份不同。
- 帕卡按传统的虚岁与性别递推，不套用汉地命卦公式。

### 其他命类体系

- 姓名学五格明确标注为日本熊崎氏近代法，不称为中国古法。
- 人类图、数字命理、玛雅卓尔金历、阿兹特克圣历和爪哇 Weton 均为简式结构盘。
- 阿兹特克历法使用历史相关性锚点复核。
- 上述异域或现代体系采用中性解读员口径，不使用道门人设，也不冒称国学正统。

## 相类

### 面相与手相

- 多模态模型先按固定观察协议输出可见特征，再映射到预设的传统分类。
- 手相包含主要线纹与 Hast Rekha 对照项。
- 输出只描述可见形态及其在传统体系中的象义，不应推断身份、疾病、族群、人格诊断或现实命运。

### 玄空风水

- 实现二十四山下卦飞星、三元龙阴阳顺逆、三元九运、旺山旺向、上山下水、双星判局及东西四命卦配宅。
- 包含兼向替卦、城门诀和流年飞星；靠近 4.5° 兼线时自动切换并提示。

已知边界：

- 不代替实地罗盘复测，也未覆盖外局砂水、水法与择日。
- 替星、城门、七星打劫等细节存在异本冲突，需要按具体师承复核。

### Vastu Shastra

- 依据空间照片或方位信息进行粗粒度特征分类。
- 当前实现不等同完整的建筑勘测、宗教仪式或专业空间处方。

## 卜类

### 易经六爻

- 按京房纳甲生成八宫归宫、六亲、伏神、月建日辰旺衰、月破、日冲、六冲六合、变爻六亲、进退神、反伏吟与应期候选。
- 日冲静爻区分暗动与日破；支持旬空出空、月破填合、入墓冲开等候选提示。
- 23 点按所选换日口径处理。

已知边界：

- 用神只做问题关键词初筛，不代替人工分辨问事类别与世应、动变、旺衰的综合判断。
- 程序不宣称生成完整师承断卦。

### 梅花易数

- 实现本卦、互卦、变卦和体用关系。
- 尚未覆盖字占、声音、外应触机等诸法；随机取象仍需使用者结合现场判断。

### 奇门遁甲

- 当前核心为时家奇门、转盘、拆补法。
- 包含三元拆补、旬空、驿马、门迫门制、六仪击刑、十干入墓、十干克应、八门九星旺相休囚、日时干落宫用神。
- 天盘伏反吟与值使门伏反吟分开标注。

已知边界：

- 置闰法和飞盘法只作审计提示，不按该法排盘。
- 茅山法、飞盘及部分克应格局尚未实现。
- 异本冲突处标为待考，不输出唯一结论。

### 大六壬

- 实现四课三传与九宗门取传：贼克、知一、涉害、见机、察微、缀瑕、遥克、八专、别责、昴星、伏吟与返吟等。
- 贵人、天将、六亲与按问事取类神并列展示。
- 收录结构条件较清晰的《毕法赋》条目；拟写断语会明确标注“拟断”，与原赋文区分。

已知边界：

- 未覆盖七百二十课全断、所有课体格局、年命上神和胎产等细法。
- 涉害深浅、贵人顺逆及特殊课体仍需要师承复核。

### 太乙神数

- 岁计式参考太乙九宫、十六神、文昌、计神、始击、主客算与大小参将，并使用立成表锚点验证。
- 可按立春或冬至岁首口径定岁计年。

已知边界：

- 岁计阳局是当前核心盘。
- 月计积月基数为类比推定，阴局为顺逆对称实验接线，均明确标为推定或待考。
- 日计、时计与阳九百六尚未完成。

### 塔罗、卢恩、Ifá、Raml 与 Sikidy

- 塔罗使用 78 张牌；卢恩使用符文石抽取交互。
- Ifá 使用奥佩雷链式随机交互与 256 奥都结构。
- Raml 与 Sikidy 依据点划或籽卜结构生成母、女、侄、证人和裁断等位置。

这些模块是简式象义盘，不宣称完整宗教仪式、师承传承或专业处方。

## AI 解读边界

- AI 只能读取固定算法已经生成的盘面上下文，不应自行改写或伪造盘面。
- 中式模块可使用“玄机子”舞台指令；异域与现代模块使用中性解读员提示词。
- 图像类模块会把所选图片发送给用户配置的模型服务商进行结构化观测；图片不会写入盘面会话存档。
- 模型输出可能出错，任何结论都应回到盘面数据、原始资料与现实证据复核。

## 复核建议

1. 先核对出生日期、时间、时区、经纬度与历法口径。
2. 对照不同换日、真太阳时、闰月或流派设置重复排盘。
3. 将“算法事实”“传统象义”“现实结论”分开记录。
4. 遇到边界日期、星座交界、兼向线或特殊格局时，使用专业历表和人工盘复核。
5. 不用任何盘面替代医疗、法律、投资、心理或安全相关的专业判断。
